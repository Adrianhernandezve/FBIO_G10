Estado final de la aplicación

Elaboración propia

Se realizó la conexión  entre el circuito desarrollado y la aplicación. Por razones desconocidas, solo logra detectar dos sesnores de los tres esperados. Sin embargo, se desarrollado un algoritmo implementado en la aplicación para la detección de señales, que se explicará a continuación.

Además, se implementó el componente "Send text", que hace que los resultados analizados sean enviados al celular del usuario como mensaje de texto. 

Elaborado en MIT App Inventor

Descripción del algortimo de detección

Una vez apse un segundo de la conexión con el sensor Bluetooth, primero se detectará el sesnsor de temperatura. Primero se verá que el valor de temperatura sea 00.00, que es el establecido por defecto. Una vez verificado esto (para evitar que los datos recibidos se sobreeescriban uno sobre otro), se verá los bytes recibidos por el sensor y el valor de temperatura asociado. De estar entre 36°C y 37,5°C, surgirá un mensaje diciendo que el estado es adecuado. Caso contrario, se mostrará un mensaje de alerta.

Un proceso similar se seguirá para los sesnores de frecuencia cardiaca y saturación de oxígeno (en ese orden). Primero se verificará que el valor anterior sea difrente al establecido por defecto (00.00). Se reciben los datos y se compara los valores numéricos con lo que se considera adecuado: para frecuencia entre 60 y 100, mientras que para SPO2 entre 90% y 100%. Según esto también saldrá un mensaje positivo o uno de alerta.
