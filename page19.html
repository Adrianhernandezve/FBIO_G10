Avances en la aplicación

Recepción de datos analógicos

Se analizó el modo de recibir datos analógicos, que es algo esencial para el correcto funcionamiento de neustro dispositivo. Ya que el circuito en Proteus todavía no está completo se elaboró una aplicación de prueba de cómo se mostrarían los datos, la cual se puede ver abajo. Esta incluye la herramienta para seleccionar al dipositivo llamando a la lista de aparatos vinculados con el teléfono.

Se usó la herramienta reloj para indicar el orden en que aparecerán los datos. Como Arduino trabaja de forma secuencial, se porgramó para que primero muestre el porcentaje de saturación de oxígeno, luego la temperatura y finalmente el ppm. La herramienta también comprobará que se esté enviando datos. Esto se puede ver en el bloque "bytes avilable to recieve > 0". De no ser así no se mostrará ningún dato.
